# Agent-Core 投资者 FAQ

## 基础问题

### 1. 你们和 Temporal、AWS Step Functions 有什么区别？
Temporal 是“编排式”的，依赖中心化数据库；Agent-Core 是“自治式”的，利用 CRDT 实现去中心化状态对齐，网络断开时可继续执行，恢复后自动收敛。

### 2. 凭什么保证 Exactly-Once？
三重锁止：HLC 保证 ID 唯一有序；Fencing Token 剥夺旧 Worker 权限；存储层原子操作确保执行与记录同生共死。

### 3. 网络断开 72 小时后恢复会发生什么？
自动愈合：各节点独立执行，恢复后 Witness 节点触发 merge，系统自动合并状态，保留合法结果，丢弃过期冲突。

## 技术深度

### 4. HLC 如何应对 NTP 回拨？
HLC 包含物理时间 `pt` 和逻辑计数器 `l`。当 NTP 回拨时，保持 `pt` 不变，递增 `l`。单毫秒内可产生 65536 个 ID，时钟停滞一秒也能处理数千万次并发。

### 5. CRDT 合并的数学证明如何落地？
Rust 类型系统强制 `merge` 操作满足半格特性（幂等、结合、交换），通过 LWW 寄存器确保所有副本最终一致。

### 6. Fencing Tokens 如何与 CRDT 协同？
存储层检查版本号，只有最新版本的提交被 CRDT 接受，确保合并时只有“最正确的执行者”留下印记。

### 7. Witness 节点部署拓扑？
建议“2+1”拓扑：AWS 和 Azure 各部署一个执行节点，第三区域（如 GCP 或本地机房）部署轻量级 Witness，构成三角仲裁。

## 未来演进

### 8. 如何适应 AI 代理自我复制？
HLC 键自动携带父代理因果信息，确保代理群体规模指数增长时，每一笔账目都可溯源到根节点。

### 9. 量子计算机出现怎么办？
UltimateKeyGen 采用插件化签名架构，预留后量子密码学算法（如 Dilithium）接口，可热更新签名组件，不改底层逻辑。

### 10. 支持 10 万节点全球部署吗？
CRDT 是无主存储，通过 Gossip 协议通信，具备线性扩展能力。

### 11. 技术壁垒能维持多久？
数学模型 + 高性能 Rust 实现的系统级整合，竞品需重构底层存储引擎，至少 18-24 个月追赶，届时我们将积累基于真实 AI 执行数据的“预测性执行”策略。
